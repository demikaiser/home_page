Python 3.7 User Guide

Reinterpret and reorganize the official Python documentation for constructing a complete language usage model.

2019/07/21

0. Introduction

It has been a while I have been Python for various purposes. As my programming skills have been improved, I became wanting to use advanced features of the language to maximize the engineering potential for projects. I tried to learn by reading and watching tutorials and guides online; however, I realized that most of them just see very small sides of the language.

What I want to have is a sound and complete mental model to be able to use the maximum spectrum of the language to fully utilize all the features. Fortunately, the official Python documentation is the place all the information exists. But the problem is it has too deep structures and doesn't prioritize by the experience levels.

This article is for my own reference, reinterpreting and reorganizing the document indices to accommodate constructing my mental model for the tool and support my programming workflow.

0. Conclusion

As I process the documentation for my personal reference, I conceived an idea that it would be nice to have these entries suggested automatically or semi-automatically by the IDE (integrated development environment). Basic linters and syntax error detectors are convenient guidance; however, they can't be navigation for the high-level options for the development process.

In the modern high-level programming environment, there is an enormous number of third-party packages available that can amp up productivity. However, engineers should be notified properly to enjoy the benefits of those packages and advanced possible idioms of the language. Wouldn't it be better to be able to supply that information to the development environment directly? I expect we can solve this in the near future.
